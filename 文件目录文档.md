## 关键文件说明

### 后端关键文件

1. **`backend/app/main.py`** - FastAPI应用入口点，包含应用初始化和路由注册
2. **`backend/app/core/config.py`** - 应用配置管理，从环境变量加载配置
3. **`backend/app/services/ai_service.py`** - AI模型服务，处理SiliconFlow和Ollama的调用
4. **`backend/app/services/knowledge_service.py`** - 知识库服务，处理向量存储和检索
5. **`backend/.env`** - 环境变量配置文件，包含数据库连接、API密钥等敏感信息
6. **`backend/app/api/v1/router.py`** - API路由配置，统一管理所有接口路由
7. **`backend/app/api/v1/endpoints/chat.py`** - 聊天相关接口实现

### 前端关键文件

1. **`frontend/src/App.vue`** - 应用根组件，包含整体布局和路由视图
2. **`frontend/src/views/Home.vue`** - 主页组件，实现聊天功能界面
3. **`frontend/src/views/Survey.vue`** - 调查表页面，实现动态表单和法律建议生成
4. **`frontend/src/views/Knowledge.vue`** - 知识库管理页面，支持文档上传和查看
5. **`frontend/src/api/chat.ts`** - 聊天API封装，处理与后端的通信
6. **`frontend/src/stores/chat.ts`** - 聊天状态管理，使用Pinia管理会话状态
7. **`frontend/src/components/KnowledgeUploadDialog.vue`** - 知识库上传对话框组件

### 配置文件

1. **`backend/.env`** - 后端环境变量配置文件
2. **`backend/requirements.txt`** - Python依赖包列表
3. **`frontend/package.json`** - 前端依赖包配置
4. **`frontend/tsconfig.json`** - TypeScript编译配置
5. **`frontend/vite.config.js`** - Vite构建配置

### 脚本文件

1. **`backend/start.bat`** - Windows后端启动脚本，自动激活虚拟环境并启动服务
2. **`backend/init_vector_store.bat`** - 知识库向量存储初始化脚本

## 数据库表结构

系统使用PostgreSQL数据库，包含以下主要表：

1. **`chat_sessions`** - 聊天会话表
   - `id`: 主键
   - `title`: 会话标题
   - `model_provider`: 模型提供商
   - `created_at`, `updated_at`: 时间戳

2. **`chat_messages`** - 聊天消息表
   - `id`: 主键
   - `session_id`: 外键关联会话
   - `role`: 角色（user/assistant）
   - `content`: 消息内容
   - `created_at`: 创建时间

3. **`survey_templates`** - 调查表模板表
   - `id`: 主键
   - `name`: 模板名称
   - `description`: 模板描述
   - `content`: JSON格式的模板内容
   - `is_active`: 是否启用

4. **`survey_records`** - 调查记录表
   - `id`: 主键
   - `template_id`: 外键关联模板
   - `answers`: JSON格式的回答内容
   - `created_at`: 提交时间

5. **`legal_suggestions`** - 法律建议表
   - `id`: 主键
   - `record_id`: 外键关联调查记录
   - `content`: 建议内容
   - `risk_level`: 风险等级
   - `created_at`: 生成时间

## 注意事项

1. 环境变量文件`backend/.env`不应提交到版本控制
2. 知识库文档支持Markdown格式，其他格式需转换
3. 向量存储目录`backend/knowledge_base/vector_store/`由系统自动生成
4. 日志文件存储在`backend/logs/`目录
5. 前端构建产物默认输出到`frontend/dist/`目录
6. 知识库文档分类：案例分析、法律文件、法律指南、合同案例
7. 启动后端服务前需确保已安装Python依赖并配置好环境变量
